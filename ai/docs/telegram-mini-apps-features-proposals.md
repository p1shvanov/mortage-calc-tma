## Рекомендуемые к внедрению

### 1. Haptic Feedback — ✅ внедрено
**Приоритет: высокий** · Сложность: низкая

- **Зачем:** тактильный отклик при действиях делает приложение ощутимо «нативнее».
- **Где:** нажатие «Рассчитать», «Добавить платёж», открытие результата, переключение вкладок на экране результата.
- **Реализация:** утилита `src/utils/haptic.ts` (impact, notification, selection); вызовы в LoanForm, MortageResult, HomePage, EarlyPaymentsForm, RegularPaymentsForm. Проверка `isSupported()` внутри.

---

### 2. Main Button — ✅ внедрено
**Приоритет: высокий** · Сложность: средняя

- **Зачем:** главная кнопка Telegram внизу экрана — привычный паттерн для TMA; можно вынести основной CTA («Рассчитать» или «К результату»).
- **Где:** на странице калькулятора — Main Button = «Рассчитать» (дублирует кнопку в форме, синхрон с `canSubmit`); на странице результата — «Изменить параметры».
- **Реализация:** mount в `init.ts`; в `LoanForm` — показ кнопки, подписка на `canSubmit` через `MainButtonSync`, onClick = submit; в `MortageResult` — кнопка «Изменить параметры» и переход на калькулятор с state.

---

### 3. Popup  
**Приоритет: средний** · Сложность: средняя

- **Зачем:** подтверждения через нативный попап Telegram вместо кастомных модалок.
- **Где:** «Удалить расчёт?» при удалении из истории; «Вы уверены?» при выходе с формы с заполненными полями (если включить жёсткую проверку).
- **Реализация:** вызов `popup.open({ title, message, buttons })` с проверкой поддержки; обработка результата по кнопке.

---

### 4. Cloud Storage (официальный API)  
**Приоритет: средний** · Сложность: средняя

- **Зачем:** единый API для облачного хранилища Telegram вместо самописной обёртки над `window.Telegram.WebApp`.
- **Где:** история расчётов в продакшене; при использовании `@telegram-apps/sdk-react` — проверить, есть ли в пакете Cloud Storage и как он монтируется.
- **Реализация:** при наличии в SDK — выделить слой хранилища (adapter), в prod использовать Cloud Storage из SDK; в dev — текущий mock/localStorage.

---

### 5. Home Screen  
**Приоритет: низкий** · Сложность: низкая

- **Зачем:** предложить пользователю добавить мини-приложение на главный экран устройства для быстрого доступа.
- **Где:** один раз после первого успешного расчёта или с экрана настроек/главной (кнопка «Добавить на главный экран»).
- **Реализация:** вызов `addToHomeScreen()` (или аналог в нашем SDK) при наличии; не показывать, если уже добавлено или метод недоступен.

---

### 6. Settings Button  
**Приоритет: низкий** · Сложность: низкая

- **Зачем:** кнопка настроек в интерфейсе Telegram открывает наше меню настроек (язык, тема, FAQ).
- **Где:** открытие экрана/секции настроек (например, с главной страницы).
- **Реализация:** mount Settings Button, по нажатию — навигация на `/settings` или открытие панели настроек (если будет отдельный экран).

---

### 7. Share / Share URL  
**Приоритет: низкий** · Сложность: средняя

- **Зачем:** «Поделиться расчётом» — отправить ссылку или текст результата в чат.
- **Где:** экран результата расчёта (кнопка «Поделиться»).
- **Реализация:** при наличии в SDK — `shareURL` или аналог; сформировать текст/ссылку (например, с параметрами в hash или коротким описанием), открыть шаринг.

## Не приоритетны для калькулятора

| Фича | Почему |
|------|--------|
| **Invoice** | Нет платёжного функционала. |
| **QR Scanner** | Не нужен для сценария калькулятора. |
| **Location Manager** | Не нужен для расчётов. |
| **Biometry** | Обычно избыточен для калькулятора без чувствительных персональных данных. |
| **Secondary Button** | Имеет смысл только при активном использовании Main Button и втором главном действии. |
| **Emoji Status** | Не влияет на UX калькулятора. |

---

## Порядок внедрения (рекомендуемый)

1. **Haptic Feedback** — быстрый выигрыш по ощущениям от приложения.
2. **Main Button** — привычный элемент TMA, усиление основного действия.
3. **Popup** — подтверждение удаления расчёта и при необходимости выхода с формы.
4. **Cloud Storage** — если в нашем SDK есть поддержка, подключить к текущему storage layer.
5. **Home Screen** — опционально, для вовлечения.
6. **Settings Button** — опционально, для доступа к настройкам из интерфейса Telegram.
7. **Share** — по желанию, для виральности и удобства.
